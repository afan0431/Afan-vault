# 环境搭建

本文档覆盖运行知识库系统所需的两个核心服务：**Ollama**（本地模型运行时）和 **Qdrant**（向量数据库）。

如果选择全程使用 OpenAI API 作为 Embedding 方案，可以跳过 Ollama 部分，只需搭建 Qdrant。

---

## Ollama

Ollama 是一个本地大模型运行时，通过统一的 HTTP 接口管理和调用各类开源模型，包括 Embedding 模型和生成模型。

### 安装

前往 [ollama.com](https://ollama.com) 下载对应平台的安装包。安装完成后，Ollama 会作为后台服务自动运行，并监听 `localhost:11434`。

支持平台：macOS、Linux、Windows（Preview）

### 拉取 Embedding 模型

```bash
# 拉取 BGE-M3（约 1.2GB）
ollama pull bge-m3

# 可选：Reranker 模型
ollama pull bge-reranker-v2-m3

# 查看已安装的模型
ollama list
```

### 验证

```bash
curl http://localhost:11434/api/embeddings \
  -d '{"model": "bge-m3", "prompt": "测试文本"}'
```

返回包含 `embedding` 数组的 JSON 即为成功。

### 跨机器访问配置

Ollama 默认只监听本机。如果 Qdrant 运行在另一台机器，需要修改监听地址：

**macOS / Linux**

在 `/etc/systemd/system/ollama.service` 中添加环境变量：
```
Environment="OLLAMA_HOST=0.0.0.0:11434"
```

**Windows**

在系统环境变量中添加 `OLLAMA_HOST=0.0.0.0:11434`，重启 Ollama 服务生效。

> ⚠️ 开放 0.0.0.0 后，任何能访问该机器的设备都能调用模型接口。内网环境下通常没有问题，公网环境需要配合防火墙限制访问来源。

## Qdrant

Qdrant 通过 Docker 运行是最简单的方式，也是最推荐的部署方式。

### Docker Compose 配置

```yaml
# docker-compose.yml
services:
  qdrant:
    image: qdrant/qdrant:latest
    container_name: qdrant
    ports:
      - "6333:6333"    # HTTP REST API
      - "6334:6334"    # gRPC
    volumes:
      - qdrant_data:/qdrant/storage
    restart: unless-stopped

volumes:
  qdrant_data:
```

```bash
# 启动
docker compose up -d

# 验证（访问管理界面）
open http://localhost:6333/dashboard
```

### 创建知识库 Collection

不同 Embedding 模型的向量维度不同，Collection 在创建后不能更改维度，**先确定 Embedding 方案再创建**。

**使用 OpenAI text-embedding-3-small（1536 维）**

```bash
curl -X PUT http://localhost:6333/collections/dev_knowledge \
  -H 'Content-Type: application/json' \
  -d '{
    "vectors": {
      "size": 1536,
      "distance": "Cosine"
    }
  }'
```

**使用 BGE-M3 混合检索（Dense 1024 维 + Sparse）**

```bash
curl -X PUT http://localhost:6333/collections/dev_knowledge \
  -H 'Content-Type: application/json' \
  -d '{
    "vectors": {
      "dense": {
        "size": 1024,
        "distance": "Cosine"
      }
    },
    "sparse_vectors": {
      "sparse": {}
    }
  }'
```

### 在远程服务器上部署

将 `docker-compose.yml` 复制到目标服务器，执行同样的命令即可。

注意事项：
- 内网使用：防火墙只开放内网 IP 段的 6333 端口
- 公网访问：必须在 Qdrant 前配置反向代理（Nginx）并启用认证
- 数据持久化：`qdrant_data` volume 会保留在服务器上，容器重建不丢数据
