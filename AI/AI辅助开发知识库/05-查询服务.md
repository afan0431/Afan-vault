# 查询服务

查询是知识库系统的使用侧。每次向 AI 提问前，先从知识库检索相关片段，将其注入 Prompt，再发给 LLM 生成回答——这就是完整的 RAG 查询流程。

---

## 查询流程

```
用户提问（自然语言）
        ↓
生成查询向量（Embedding API）
        ↓
Qdrant 向量检索 → Top 20 候选
        ↓
（可选）Reranker 精排 → Top 5
        ↓
构建 Prompt：系统指令 + 知识片段 + 用户问题
        ↓
发送给 LLM → 返回回答
```

---

## 查询脚本

```python
# query_knowledge.py
import os
import httpx

QDRANT_URL = os.environ.get("QDRANT_URL", "http://localhost:6333")
COLLECTION = "dev_knowledge"
OPENAI_API_KEY = os.environ.get("OPENAI_API_KEY", "")


def get_embedding(text: str) -> list[float]:
    resp = httpx.post(
        "https://api.openai.com/v1/embeddings",
        headers={"Authorization": f"Bearer {OPENAI_API_KEY}"},
        json={"input": text, "model": "text-embedding-3-small"},
        timeout=15,
    )
    resp.raise_for_status()
    return resp.json()["data"][0]["embedding"]


def search(query: str, top_k: int = 5, category: str = None) -> list[dict]:
    """向量检索，支持按 category 过滤"""
    vector = get_embedding(query)

    body = {
        "vector": vector,
        "limit": top_k,
        "with_payload": True,
        "score_threshold": 0.6,  # 过滤低相关度结果
    }

    if category:
        body["filter"] = {
            "must": [{"key": "category", "match": {"value": category}}]
        }

    resp = httpx.post(
        f"{QDRANT_URL}/collections/{COLLECTION}/points/search",
        json=body,
        timeout=10,
    )
    resp.raise_for_status()
    return resp.json()["result"]


def build_prompt(query: str, chunks: list[dict]) -> str:
    """构建注入知识的 Prompt"""
    if not chunks:
        context = "（知识库中未检索到相关内容）"
    else:
        parts = []
        for i, chunk in enumerate(chunks, 1):
            p = chunk["payload"]
            parts.append(
                f"[{i}] {p.get('title', '未命名')}\n"
                f"分类：{p.get('category', '')} | 来源：{p.get('source', '')}\n\n"
                f"{p.get('content', '')}"
            )
        context = "\n\n---\n\n".join(parts)

    return f"""你是一个熟悉该项目的开发助手。请**严格基于以下知识库内容**回答问题。

如果知识库内容不足以回答，请明确说明"知识库中没有相关信息"，不要凭空推断。
引用知识库内容时，请标注来源编号，如 [1]。

## 项目知识库

{context}

## 问题

{query}"""


def ask(query: str, category: str = None) -> str:
    chunks = search(query, top_k=5, category=category)
    prompt = build_prompt(query, chunks)

    resp = httpx.post(
        "https://api.openai.com/v1/chat/completions",
        headers={"Authorization": f"Bearer {OPENAI_API_KEY}"},
        json={
            "model": "gpt-4o-mini",
            "messages": [{"role": "user", "content": prompt}],
            "temperature": 0.1,
        },
        timeout=60,
    )
    resp.raise_for_status()
    return resp.json()["choices"][0]["message"]["content"]


if __name__ == "__main__":
    import sys
    query = " ".join(sys.argv[1:]) if len(sys.argv) > 1 else input("问题：")
    print("\n" + ask(query))
```

### 使用示例

```bash
python query_knowledge.py "项目如何处理 CQRS 中的命令返回值？"
python query_knowledge.py "订单状态可以随意切换吗？"
python query_knowledge.py "数据库排序有哪些注意事项？"

# 限定知识类型
```

## Prompt 设计要点

### 明确指令限定来源

```
❌ "你是项目专家，请回答..."
   → AI 会用训练记忆填充，可能与项目实际不符

✅ "请严格基于以下知识库内容回答，知识库外的内容请说明不知道"
   → 严格限定回答来源，显著减少幻觉
```

### 要求引用来源

在 Prompt 中要求 AI 标注引用来源（`[1]`、`[2]`），有两个好处：
1. 方便核查 AI 的回答是否基于真实检索到的内容
2. 当知识库内容有错误时，可以快速定位到对应文件

### 温度参数建议

```python
temperature = 0.1   # 知识查询场景，需要准确，不需要创意
temperature = 0.3   # 代码生成场景，允许少量变体
```

## 与 AI 工具的集成方式

### 方式一：命令行检索，手动复制

最简单的方案——开始一项开发任务前，先跑一次检索，把相关知识片段复制到对话的开头作为背景信息。

### 方式二：MCP Server（推荐）

将查询服务封装为 MCP Server，Claude Code 等支持 MCP 的 AI 工具可以在对话中自动调用知识库。

```python
# mcp_server.py
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent
import asyncio

app = Server("dev-knowledge")


@app.list_tools()
async def list_tools():
    return [
        Tool(
            name="search_dev_knowledge",
            description="搜索项目开发知识库，获取架构决策、业务规则、已知陷阱等信息",
            inputSchema={
                "type": "object",
                "properties": {
                    "query": {"type": "string", "description": "搜索问题"},
                    "category": {
                        "type": "string",
                        "enum": ["architecture", "rule", "term", "pitfall", "contract"],
                        "description": "可选，限定知识类型",
                    },
                },
                "required": ["query"],
            },
        )
    ]


@app.call_tool()
async def call_tool(name: str, arguments: dict):
    if name == "search_dev_knowledge":
        chunks = search(arguments["query"], category=arguments.get("category"))
        result = build_prompt(arguments["query"], chunks)
        return [TextContent(type="text", text=result)]


async def main():
    async with stdio_server() as (read, write):
        await app.run(read, write, app.create_initialization_options())


asyncio.run(main())
```

安装 MCP 依赖：

```bash
pip install mcp
```

在 AI 工具的 MCP 配置中注册这个服务后，AI 在处理开发相关问题时可以主动调用 `search_dev_knowledge`，无需人工干预。

## 调试技巧

### 查看检索质量

```python
chunks = search("你的问题")
for chunk in chunks:
    print(f"分数：{chunk['score']:.3f} | 标题：{chunk['payload']['title']}")
```

分数解读：
- `> 0.85`：高度相关
- `0.7 ~ 0.85`：中等相关，通常有用
- `< 0.6`：低相关，通常是噪音（已被 score_threshold 过滤）

### 常见问题排查

| 现象 | 可能原因 | 解决方式 |
|------|---------|---------|
| 检索不到明明存在的知识 | score_threshold 设置太高 | 适当降低阈值，观察分数分布 |
| 检索结果有大量不相关内容 | 知识文件粒度太粗 | 将大文件拆分为更细的知识点 |
| AI 给出了知识库中没有的内容 | Prompt 指令不够严格 | 加强"知识库外内容说不知道"的指令 |
| 分数普遍低于 0.5 | 知识库缺乏该领域内容 | 补充对应知识文档 |
